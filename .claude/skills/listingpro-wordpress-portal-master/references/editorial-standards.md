# Editorial Standards für Campus Sparbuch und Claude Cowork

## Ziel

Schreibe präzise, operativ und ohne KI-Floskeln. Campus-Sparbuch-Ausgaben müssen von Geschäftsführung, Entwicklern, Vertrieb, Content-Team oder Browser-Operator direkt genutzt werden können. Jede Aussage braucht Funktion: Entscheidung, Umsetzung, Prüfung oder Übergabe.

## Grundregeln

| Regel | Umsetzung |
|---|---|
| Direkt schreiben | Keine Ankündigungssätze. Mit Inhalt beginnen. |
| Akteure nennen | Aktive Stimme nutzen. Wer tut was? |
| Spezifisch werden | Keine vagen Aussagen über „Bedeutung“, „Potenzial“ oder „Struktur“, ohne die konkrete Konsequenz zu nennen. |
| Annahmen trennen | Fakten, Annahmen, Risiken und Empfehlungen getrennt ausweisen. |
| Keine Floskeln | Keine generischen Business-Phrasen, keine Motivationssprache. |
| Keine Schein-Dramaturgie | Keine künstlichen Kontraste, keine rhetorischen Fragen, keine Punchline-Absätze. |

## Verbotene Einstiege und Füllphrasen

Streiche diese Muster. Schreibe den Punkt direkt.

| Vermeiden | Besser |
|---|---|
| „Hier ist der Punkt“ | Direkt mit dem Punkt beginnen |
| „Lass mich klar sein“ | Aussage ohne Ansage schreiben |
| „Die Wahrheit ist“ | Fakt oder Einschätzung benennen |
| „In der heutigen Landschaft“ | Konkreten Markt, Kanal oder Prozess nennen |
| „Es ist wichtig zu beachten“ | Nur nennen, was für Umsetzung oder Entscheidung relevant ist |
| „Am Ende des Tages“ | Konsequenz benennen |

## Business-Jargon ersetzen

| Vermeiden | Verwenden |
|---|---|
| navigieren | handhaben, lösen, steuern |
| Deep Dive | Analyse, Prüfung |
| Gamechanger | relevanter Hebel, wenn begründet |
| Landscape | Markt, Umfeld, Situation |
| Circle back | erneut prüfen, zurückkommen |
| Moving forward | nächster Schritt, ab jetzt |
| On the same page | abgestimmt, geklärt |

## Strukturmuster vermeiden

Keine Formeln wie „Nicht X, sondern Y“. Schreibe Y direkt. Keine negativen Aufzählungen vor der eigentlichen Aussage. Keine künstliche Fragmentierung wie „Ein System. Kein Projekt. Ein Hebel.“ Schreibe vollständige Sätze mit Subjekt, Verb und Objekt.

## Handoff-Stil

Ein gutes Handoff enthält Status, Kontext, konkrete Aufgabe, Dateien, Kommandos, Risiken, Akzeptanzkriterien und nächste Handlung. Es enthält keine Höflichkeitsumwege und keine Kommentare wie „erledigt“, wenn die Antwort direkt an Claude Cowork weitergegeben werden soll.

## Prüfcheck vor Ausgabe

| Frage | Beste Antwort |
|---|---|
| Kann Jesse den Text direkt weiterleiten? | Ja, ohne Erklärung daneben. |
| Kann ein Entwickler daraus handeln? | Ja, mit Datei-, Pfad- oder Kommandoangaben. |
| Sind Fakten und Annahmen getrennt? | Ja. |
| Enthält der Text Füllphrasen? | Nein. |
| Gibt es einen klaren nächsten Schritt? | Ja. |
